# 笔记

  • https://learnopengl-cn.github.io/intro/

    这个页面的布置风格，配色，标题下浅浅的分割线，各种段间距的方式，以及整个陈述的条理可以学习。

  • OpenGL本质上是一个状态机，比如当我们想要在屏幕上画一条直线的时候，不是去规定起点、终点、长度等，而是通过某种方式去改变状态机中的各个参数，使得整个程序处于某个状态，类似人类的某个姿势，然后执行一次绘画的操作，那么在这个姿势的情况下，画出来的就是一条直线。

  • 改变参数的方法是这样的。比如我们要绘制一个800*600的屏幕，那么我们先生成一个id，然后创建一个对象，和id绑定，这个对象会被存储在额外的地方，然后将对象放到状态机中可以改变屏幕分辨率的地方，对对象进行某种操作，达到改变分辨率的效果，注意，此时对于对象的改变，会影响状态机其他部分的参数。之后将嵌入对象部分的状态机的id重新设置为默认的0，也就是相当于取出了对象，恢复默认。之后当要使用这一设置的时候，将id导入便可，相当于自动重新做了一次屏幕分辨率的设置。

    状态称为context，即上下文。

  • 在OpenGL中，任何事物都处在3D空间中，但屏幕是2D的，这导致大部分工作是将3D坐标转化为2D，用到的是渲染管线，即管线，实际上是一堆原始图形数据经过一系列的函数处理出现在屏幕上的过程。

    管线的工作分两部分，一是3D转2D，二是给2D坐标配上颜色像素。

    这些阶段中的函数的一大特点是并行性，意味着显卡的成千上万的小核心可以并行操作，这些并行操作的小程序叫着色器Shader。

  • 我们在输入顶点着色器的时候，用的是VBO（vertex buffer object），即顶点缓存对象，因为CPU到GPU的传输速度很慢，所以有必要一次性传大批数据。但我们调用VBO的时候，每一个顶点都会指定一个新的指针，这将消耗大量资源，实际的做法是用VAO（Vertex Array Object），注意，这并不是VBO并列的一个概念，将VBO需要用的顶点属性配置到VAO中，VAO是一个数组，因此大量的数据只需要一个指针，并且这样也方便不同顶点之间的切换。这意味着只需要调动那一堆指针一次。

    比如绘制立方体，就拿出立方体对应的VAO，绑定它，绘制，然后解绑。

  • 当我们绘制矩形的时候，使用的是2个三角形叠加，但这样会有6个顶点，额外的开销是巨大的。因此有了索引缓冲对象IBO/EBO（Index/Element Buffer Object），它指定所有点的绘制顺序，并且会过滤掉重复的点。绑定VAO的时候会同时绑定EBO。

  • # 为什么一张带有颜色的图可以指定材质的凹凸？

    颜色是由某种颜色着色器操作的，凹凸是由某种凹凸（位置）着色器操作的，着色器的特点是有输入和输出，比如凹凸，其显然需要的输入是各个顶点的坐标。在OpenGL中，坐标由向量表示，向量可以有1，2，3，4维，表示空间的是xyzw四维，而颜色也是用顶点着色器表示，有RGBA四维，所有向量都经过某种归一化，即值在绝对值1之间，因此，颜色着色器中的顶点数据可以完美映射到表示凹凸的顶点着色器中。

    这也是为什么凹凸贴图呈3紫色的原因，因为坐标系纵向是y轴，对应的是绿色通道，而凹凸一般来说竖直方向的变化总没有水平的变化大，因此颜色就集中在RB两色的排列组合上，可以看到颜色集中在紫色附近。另外猜测，可能x，y方向并没有对应到rgb值的0点，也就是说，取值可能在155-255之间，因此不会出现明显的大红色这样的。

  • 纹理存在的意义是可以避免用大量的顶点来表示细致的颜色分布，否则会造成大量的开销。

    只需将纹理的某些点对应顶点，然后进行插值运算即可。

    PS：

    ​ 纹理还可以用来存储大量的数据。

  • 一个光源对它的ambient、diffuse和specular光照有着不同的强度。环境光照通常会设置为一个比较低的强度，因为我们不希望环境光颜色太过显眼。光源的漫反射分量通常设置为光所具有的颜色，通常是一个比较明亮的白色。镜面光分量通常会以最大强度发光。

  • 高光贴图（specular），决定比如木头和金属对镜面光的反射能力，就是那个白圈，所以用黑白图表示反射强度的大小。

    可以用PS，将漫反射纹理转换为镜面光纹理将图像转换为黑白的，并增加亮度/对比度就好了。

    若高光贴图的颜色非黑白，相对的就设置了高光的颜色，但是现实世界的高光颜色大都是由光源本身决定的，额外的设置颜色会不真实。

  • 在第一个三角形绘制完之后，每个后续顶点将会在上一个三角形边上生成另一个三角形：每3个临近的顶点将会形成一个三角形，只需要以正确的顺序生成3个相连的三角形就行了。

  • 实例化的作用是这样的。比如画一百根草，每根草虽然只有几个三角面，我们也可以一次性将一堆顶点数据发送到GPU，但是，告诉GPU控制顶点数据需要消耗很多资源，比如告诉GPU从哪个缓冲读取数据，这些工作董事在CPU-GPU总线上进行的，所以，即便渲染顶点很快，但告诉它怎么渲染却很慢。

    实例化是一下子让它照原样渲染100个，但是在渲染的时候，让它乘以某个矩阵，实现位移或者旋转。

    因此，实例化渲染会大大加强显卡的渲染能力，避免卡在CPU-GPU传输数据速度的瓶颈上。

  • 多重采样MSAA（原理并不是理解得很清楚，不看源代码恐怕解释不了）

    在光栅话的过程中，如果以正方形的某个点作为判断是否覆盖的标准，会产生明显的锯齿，多重采样便是一种解决方式。

    这种方式可以引申到像素画边缘过度色块取色的原理。

    但是！

    问题在于，计算机并不会知道哪里是边缘，具体的图像也远比这个三角形复杂，边缘将会有很多，也更弯曲。最初始的SSAA便是渲染出数倍于屏幕输出的采样点来做平均，开销太大。注意，即使你不把颜色渲染到屏幕上，但计算平均色这一任务本身，需要调用四个子节点的颜色信息，也就是说他们本身必须已经有了颜色信息，已经被"着色"了的，

    MSAA本质上是解决了那些被全覆盖和全不覆盖的像素的资源占用问题，捎带的对于百分比占用的像素也会有性能提升。

    它的工作方式是，提取的是子节点是否被覆盖的信息，然后计算出像素中被覆盖的百分比，用顶点颜色乘以百分比来对像素整体着色。因此对于每个像素，只着色一次。

    而计算是否被覆盖，是在光栅化阶段就可以完成了的。看上去似乎在光栅化阶段对子节点计算和在着色阶段计算，开销相差可能并不是特别的大，但是，至少，在实际应用中，shading的运算量远大于光栅化，因此节省的开销是十分可观的。

  • 关于漫射和高光

    漫射可以理解为物体本身的颜色，眼睛看到的颜色是光源强度和光线到平面的方向矢量与平面法线矢量点乘的结果，也就是说，无论从哪个角度看，只要物体和光源固定，漫射的颜色和强度是一定的。也就是说，漫射其实是一个平面没有实际碎片化但理解成碎片化乱射光线后的统计结果。

    高光是另一个反射的概念，从光源发出的光，会在平面上反射，在此，平面没有被模拟成碎片化，法线的存在只是为了计算反射的方向矢量，在这一确定的方向上，光强等于入射的光，一般都为白色。但是对于观察者，高光会因为观察的角度不同而变化，比如玻璃球的边缘看上去特别亮，而中间就很透明，反射不强烈。所以需要用视线的方向矢量和反射光的方向矢量做点乘，此处为了效果，会有一些数学额外的处理，好像是32次幂之类，我也不知道确切的功效，反正最终的结果就是，如果视线和反射光的角度大，那么表现出来的就是高光的强度变低。

  • Phong和Blinn-Phong的区别

    https://learnopengl-cn.github.io/05%20Advanced%20Lighting/01%20Advanced%20Lighting/

    本质上是为了解决当视线与反射角度大于90度时的情况，这时的函数会把高光设为0，一般来说，这时候问题不会太大，但是当物体本身的漫反射能力较低时，这样的高光便不能被忽略了，注意，高光并不像漫反射一样，超过90度就不行了的，漫反射是因为超过90度就到背面去了。

    Blinn-Phong就是为了解决这个问题的，方式是不再计算视线与反射光的夹角，而是新设了一个半矢量，计算入射与视线矢量的合成，然后用这个合成去和法线比较。二者的区别基本就这么点。

  • 如CRT显示器，两倍的电压并不是产生两倍的亮度，二者关系越是2.2次幂，这叫做监视器Gamma。

    Gamma也叫灰度系数，每种显示设备都有自己的Gamma值，都不相同，有一个公式：设备输出亮度 = 电压的Gamma次幂，任何设备Gamma基本上都不会等于1。

    但是，人类所感知的亮度正好和CRT显示出的差不多，人差不多是2次幂，比如1份光子，亮度加倍需要四份光子。

    但是，比如（0.5，0，0）表示暗红色，0.5^2.2=0.218，若要暗红色亮度翻倍，线性变为1，实际在显示器上的亮度为4.5倍，虽然人眼看上去是翻倍效果，但是要知道，在真实场景中，比如打上两个光，真实的效果是光子数翻倍，而不是人眼的感觉翻倍。

    因此，原本线性的颜色，在显示到监视器上的时候，需要进行处理，即^(1/2.2)

    基于Gamma2.2的颜色空间叫做sRGB颜色空间。每个监视器的gamma曲线都有所不同，但是Gamma2.2在大多数监视器上表现都不错。

    但是，这种矫正，只有CRT是2.2的，如果一直用同样的矫正方式，对于尤其是液晶显示这样明显不同的显示器，就会出现很不真实的颜色。因此Gamma矫正的方式并不是固定和通用的。

    不是所有纹理都是在sRGB空间中的，所以当你把纹理指定为sRGB纹理时要格外小心。比如diffuse纹理，这种为物体上色的纹理几乎都是在sRGB空间中的。而为了获取光照参数的纹理，像specular贴图和法线贴图几乎都在线性空间中，所以如果你把它们也配置为sRGB纹理的话，光照就坏掉了。

# 在真实世界中，光线强度的衰减是平方反比，但是这样的衰减太强烈，看上去会不真实，实际用的是一次反比函数。原因在于屏幕上出现的并不是线性空间。显示在监视器上的亮度实际变成了(1.0/distance)^2.2，和真实相符。当然这是在不用Gamma修正时的。

  • Gamma矫正的优势便是让你可以在线性空间中操作，更符合客观世界的感觉。

  • 深度值即从光源的透视图下见到的第一个片段，存储这些深度值的纹理即阴影贴图。

  • 法线贴图和凹凸贴图一个道理，将法线的xyz换成rgb，因此法线贴图看起来是蓝色调的，因为大部分法线都是偏向Z轴的。（这个说法不对，正确的在这里https://www.zhihu.com/question/23706933）应该是由（-1，1）映射到（0，1），通过x=（x+1）/2来实现，因此没有偏扰的会是（0.5，0.5，1）

    法线贴图可以这么理解，比如一个砖块，我们建模成长方体，贴图，我们会将长方体分割成许许多多的小面，但是不可能对每个面改变它们的顶点位置来适应光照，从而产生真实的颗粒效果，否则这样的数据输入太可怕了，需要一各个调整小面，也根本没办法建模，所用的办法便是法线贴图，使得原本都是垂直于小面的法线，有了方向的偏移，在小面实际数据不改变的情况下，模拟出面的朝向的改变，因为所有光线的计算都是基于法线的，而不管你面的顶点是什么情况。而肉眼只会分辨这些折射出来的光的总体效果，而不可能去细看每一个小面的具体朝向是否有bug。

    所谓切线空间，因为我们获得法线不是目的，而是为了之后的光线计算，切线空间的z轴即平面真实的法线方向。计算光线是在切线空间中，这样同一张贴图，在对正方体不同面渲染的时候，因为切线空间的不同，便会有不同的效果。

# 视差贴图，即凹凸贴图，H（A）=P，黄色为光线方向，实际应该的物体轮廓是红线，因此看到的应该是B点，但我们不用置换，光线是照射到黑色的平面上的，实际上看到的是A点，那么需要将A点的纹理换成B点的纹理便可，方法是取得H（P）这一处的纹理信息，显示在A处。但这样倘若AB纵向落差太大，会有明显误差。

# HDR(High Dynamic Range, 高动态范围)，并非是贴图的别称！

他是为了解决这么个问题，比如，场景中有几个光源叠加，使得光强超过了1，那么他们会被限制为1，但这样就会导致失去了很多细节，很多原本应该不同的区域都糊在了一起。解决方式并不是调整光强，而是允许合颜色贴图暂时超过1，使得我们有了更大的颜色范围，最后再转换到1以内。这种方法可以保护住非常亮和非常暗处的细节。可以让亮的东西非常亮，暗的东西非常暗。

DR原本只是被运用在摄影上，摄影师对同一个场景采取不同曝光拍多张照片，捕捉大范围的色彩值。这些图片被合成为HDR图片，从而综合不同的曝光等级使得大范围的细节可见。

这和人眼的机理很类似，比如一个场景中同时有极亮和极暗的地方，那么看亮处瞳孔缩小，暗处瞳孔放大，都可以看清细节，但是对于一张照片，其曝光度是一定的，即瞳孔大小不可调，因此亮暗处的清晰度是个矛盾，需要多曝光度照片的合成。

转换HDR值到LDR（Low，即转换到1一下）的过程叫做色调映射(Tone Mapping)，注意并不是简单乘个系数，其算法的目的是为了保持更多的细节。

最简单的色调映射算法是Reinhard色调映射，它涉及到分散整个HDR颜色值到LDR颜色值上。

HDR真正的优势在于复杂的场景和光照可以被显示出来。

# bloom泛光，即光晕，为了解决比如白纸和太阳看起来一样的问题。

高斯模糊可以解决这个问题，比如一个8* 8方块，以8*8的精度模糊，每个方块的颜色需要采样64次，这是个极大的开销，然而高斯分布有个特性，可以拆解为UV两个方向，即第一步横向展开为8单位的像素，然后每个像素再纵向展开，这样即使在不使用额外缓存的情况下，每个方块也只需要采样8+8次。

  • 粗糙度是统计学的概率问题，基于粗糙度来计算出某个向量的方向与微平面平均取向方向一致的概率。这便是视线与光线一半的那个向量。

    粗糙平面反射昏暗是因为要保持能量守恒，粗糙平面镜面反射的区域会增大。

    镜面反射即反射光，而漫反射是折射进去被吸收过后反出来的光，即颜色。

    次表面散射，即光线折射进物体后，进过吸收再反出来的时候，并不是一束光线了，而是折射进去之后，经过了各种碰撞，已经分散成多束光了。

    之所以软件选项会区分金属和非金属，是因为金属的折射光会被全部吸收，不会漫反射出来。

  • 渲染方程，即多个光源对于空间某一平面叠加的辐射总能量公式，参见https://learnopengl-cn.github.io/07%20PBR/01%20Theory/，讲得无比好。

    式中，L是辐照度，n是发现，是立体角，是半球面积，f是一个限制，即BRDF（Bidirectional Reflective Distribution Function），他的作用是基于表面材质属性来对入射辐射率进行缩放或加权。双向指的是反射光和透射光两部分返出来的都要计算。

    BRDF，他接受入射光方向，出射观察方向，法线n，以及一个用来表示平面粗糙度的参数a作为函数的输入参数。可以近似求出每束光线对一个给定了材质属性的平面上最终反射出来的光线所做出的贡献程度。

    比如，一个完全平整的平面，那么只有一束光的BRDF会不为0，实际上是1，而其余所有的入射光线的BRDF都为0。

  • 漫反射模型的开销非常大，一般使用Lambertian模型效果就很好了。

  • GGX 2007年提出

    https://blog.uwa4d.com/archives/1582.html

    其目标解决的问题是，如何将微表面反射模型推广到表面粗糙的半透明材质，从而能够模拟类似于毛玻璃的粗糙表面效果。GGX便是文章中提出的新的微表面分布函数。

    GGX是由几个函数组合而成，其中一个描述了微表面的分布，不同的分布函数造成表面不同程度的粗糙度。

  • 控制反射亮斑大致是这么个原理。比如，如果平面是彻底平的，那一束光线照射过来，反射出来的光就是一个点，没有分散，特别亮和小，但如果，一个平面粗糙，即很多微平面取向不一，那么就会有很多平面收不到这种直接的反射光，相当于不在反射角上观察时的效果，可以参加高光计算的方式，也就是说，这个点只有一部分是被完全点亮的，还有一部分暗了。此外，这个点还会扩大，因为微平面取向各异，导致这个大平面周围的其他平面的微平面也有一定比例会恰好处在使反射光进入观察者视角的情况，也同样表现出高亮，但比例显然会降低，这样便慢慢扩散出去，可以到很远。

  • BRDF常用的是Cook-Torrance BRDF，主要含有三个函数，DFG

    D（Normal Distribution Function）：正态分布函数，用来表示粗糙度

    F（Fresnel quation）：菲涅尔函数，描述不同角度反射光线所占比率

    G（Geometry Function）：描述微平面间相互遮蔽的比率，这种遮蔽会损耗光线能量，也是用粗糙度作为输入参数

  • 对于导体表面来说，菲涅尔的基础反射率在0.5以上，而非金属在0.04，由于金属会吸收折射光，没有漫反射出来，因此没办法表示物质的颜色，所以他们的基础反射率一般是带颜色的，而非金属不带。用参数金属度来描述材质是金属还是非金属。

    Cook-Torrance BRDF

    这便是我们现在一般意义上说的PBR。

    • 反照率（albedo）：

      为每一个金属纹理像素指定表面颜色或者基础反射率，类似漫反射纹理，但是反照率纹理只有颜色，而漫反射纹理中常包含一些细小的阴影或者深色的纹理。

    • 环境光遮蔽（AO）：

      比如如果我们有一个砖块表面，反照率纹理上的砖块裂缝部分应该没有任何阴影信息，但AO贴图则会把那些光线较难逃逸出来的暗色边缘指定出来。

  • 基于图像的光照(Image based lighting, IBL)，即光源不是可分解的直接光源，而将整个环境视为一个大光源。这是一种更精确的环境光照输入格式。

    相当于将贴图上的所有点都当做一个点光源，来计算所需位置上的辐照度。但这种方法还有另一个要求，即解决积分需要快速和实时。以反射项为例，所以采取这么一个办法，即不妨假设物体在场景中央，即半球积分的球心处，计算出从球心射向半球面的所有方向矢量上的辐照度，这里采用的是卷积的方式，即理解为场景中可以击中球心的所有表面造成的漫反射光的预计总和。

  • PBR和HDR

    基于图像的光照，只能在LDR低动态范围，因为颜色的图像值只能介于0-1之间。

    辐射度文件格式是.hdr，但用了一个技巧来存储每个浮点值，它并非直接存储RGBA通道的32位数据，而是每个通道存储8位，再以alpha通道存放指数，虽然会导致精度损失，但是非常有效。需要程序将他们重新转换成浮点数等效值。

    平面纹理的投影方式是等距柱状投影，就是古老地图的投影方式。但直接用等距柱状贴图开销很大，需要转换成立方体贴图。

    基于图像的光照，只需一次立方体贴图，大部分的工作量在于将环境贴图预计算或卷积成辐照度图。

  • 关于图像卷积。https://www.zhihu.com/question/22298352

    首先从一维了解卷积的意思，比如有一个连续信号，敲打在某个点，效果是将某一处敲击下去，信号的强度时时在变，而同时，材料也会有反弹效果，会时时想要恢复平衡位置，那么问题是，一秒的时候，材料的状态是什么样的。

    离散的卷积是这样的，将一秒分成十个点，信号在这十个点上的强度是已知的，将其乘以时间持续时间即0.1s，可以算出每个强度的实际作用效果。但是效果会随时间衰退，衰退曲线便是材料的回弹的曲线，越早产生的信号，恢复的时间越长，在曲线上的位置也越靠后，将信号和回弹曲线对应的点相乘，最后求和便是材料的状态。

    将这个概念运用到图像中去，比如一个环境光，如果表示其照亮的某一点的亮度呢？在环境光贴图上的每一个点，可以看做信号的输入，每一个点的颜色亮度即信号的强度，而该点到目标点之间因为角度等的原因导致的衰退，便是衰退曲线，相当于用每个点来替代每个时刻来排列在强度曲线和衰退曲线上，然后依然利用卷积的概念相乘，最后便成为目标点光照的强度了。

    一类是所求解的问题本身具有内在的随机性，另一种类型是所求解问题可以转化为某种随机分布的特征数，比如，假设你有一袋豆子，把豆子均匀地朝这个图形上撒，然后数这个图形之中有多少颗豆子，这个豆子的数目就是图形的面积。

    注意，某些蒙特卡洛方法是无偏的，即只要不断提高精度，最终都会趋向精确解，但某些蒙特卡洛方法是有偏的，不会趋向精确解，但优势是可以更快速地趋向解。

  • 重要性采样

    比如某一粗糙度的材质，光滑，那对于反射光的强度的影响，只有比较特定角度的光源贴图才会有效果，而贴图的其他部分的效果很小很小，那么此时可以通过对这一特定区域的贴图加大采样力度，而减少其他区域的采样来加快近似计算的速度。重采样分布有时候都不需要很复杂，一个简单的三角形曲线可能就会大大增加效率。

    在此处的具体应用中，即只在某些区域生成采样向量，该区域围绕微表面半向量，受粗糙度限制。
